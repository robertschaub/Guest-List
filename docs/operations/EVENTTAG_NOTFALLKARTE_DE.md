# Eventtag-Notfallkarte

## Sofortregel

Wenn unklar ist, ob die App richtig synchronisiert: **Check-in stoppen, Admin oder verantwortliche Person holen, keine Mehrfachklicks.**

## App zeigt `Offline`

1. Check-in auf diesem Gerät sofort stoppen.
2. WLAN/Mobilfunk prüfen.
3. Seite neu laden und Eventname, Datum und `Online` prüfen.
4. Wenn weiter offline: CSV- oder Papier-Backup verwenden.
5. Später nur kontrolliert mit Admin abgleichen. Keine parallelen Offline-Listen mehrerer Geräte zusammenführen, ohne Duplikate zu prüfen.

## Falscher Event im Header

1. Nicht weiter einchecken.
2. Richtigen offiziellen Event-Link neu öffnen.
3. Header prüfen.
4. Bei bereits erfolgten Check-ins Admin informieren.

## Gast nicht gefunden

1. 2-3 Buchstaben des Namens suchen.
2. Guest ID suchen, falls vorhanden.
3. Kategorie und Kommentar prüfen.
4. Wenn weiterhin kein Treffer: Admin oder verantwortliche Person fragen.

## Doppel-Check-in-Warnung

1. Nicht erneut klicken.
2. Gastdaten und Check-in-Zeit prüfen lassen.
3. Admin entscheidet über Korrektur oder Ausnahme.

## App langsam oder reagiert nicht

1. Warten, nicht mehrfach klicken.
2. Verbindung prüfen.
3. Seite neu laden.
4. Danach Eventname, Datum und Rolle erneut prüfen.

## Backup-Fallback

- Vor Eventstart muss ein aktueller Export `Alle Gäste CSV` vorhanden sein.
- Bei längerem Ausfall kann diese Liste als Notfall-Liste dienen.
- Nach Wiederherstellung der App gleicht nur ein Admin Änderungen kontrolliert ab.
